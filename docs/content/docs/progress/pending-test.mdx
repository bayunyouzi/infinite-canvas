# 待测试

- 生图工作台提示词区：新增“随机提示词”按钮和“二次元”开关；点击后会调用内置 `grok-4.20-0309-non-reasoning` 生成单行英文安全生图提示词并自动填入输入框，且提示词固定围绕美少女主题，明确覆盖画面主体、风格、构图、光线和用途；前端已修复流式回调被二次累加导致的前缀重复问题，仍需人工验证写实 / 二次元两种随机提示词是否都能稳定返回，且不再出现逐词累积、解释性废话或脏输出。
- 生图工作台生成容错：当接口返回 `Internal server error` 或常见 5xx / 空图片结果时，单张任务会自动持续重试直到成功；界面会显示每张图的重试次数，并可手动停止某张图的自动重试；仍需人工验证多张并发生成时的自动重试表现、停止重试交互与最终成功率。
- 生图默认配置：新增隐藏的内置生图渠道，默认走 `grok-imagine-image-lite`，使用内置 Base URL 和 API Key，仅作为兜底默认值；“模型”Tab 已恢复生图模型可选项与默认生图模型设置，且生图模型列表不再沿用视频/文本那套关键字过滤，用户手动填写的自定义图片模型也能保留并被选中；仍需人工验证生图页与画布生图是否既能命中内置默认配置，也能正常切换到用户自定义渠道。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
- 视频生成：修复 OpenAI 兼容视频接口在上传参考图时错误强制写入 application/json 请求头的问题；旧版依赖 `input_reference[]` 的 multipart 提交已恢复正常。
- Grok 视频 1.5：`grok-imagine-video-1.5-720p / 1080p` 改为走文档要求的异步 `POST /videos/generations` JSON 模式，创建阶段返回 `request_id`，轮询阶段读取 `GET /videos/{request_id}` 的 `status` 与 `video.url`；若渠道未开放该端点，会明确提示检查视频网关。
