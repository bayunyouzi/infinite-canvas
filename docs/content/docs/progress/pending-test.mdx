# 待测试

- 生图工作台提示词区：新增“随机提示词”按钮和“二次元”开关；点击后会调用内置 `grok-4.3-fast` 生成安全的成年女性生图提示词并自动填入输入框，仍需人工验证写实 / 二次元两种随机提示词是否都能稳定返回。
- 生图默认配置：新增隐藏的内置生图渠道，默认走 `grok-imagine-image-lite`，使用内置 Base URL 和 API Key，仅用于生图；配置弹窗不再暴露该公用渠道与生图默认模型设置，仍需人工验证生图页与画布生图是否正常命中该默认配置。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
- 视频生成：修复 OpenAI 兼容视频接口在上传参考图时错误强制写入 application/json 请求头的问题；旧版依赖 `input_reference[]` 的 multipart 提交已恢复正常。
- Grok 视频 1.5：`grok-imagine-video-1.5-720p / 1080p` 改为走文档要求的异步 `POST /videos/generations` JSON 模式，创建阶段返回 `request_id`，轮询阶段读取 `GET /videos/{request_id}` 的 `status` 与 `video.url`；若渠道未开放该端点，会明确提示检查视频网关。
