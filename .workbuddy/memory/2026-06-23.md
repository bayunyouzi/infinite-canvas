## 排查 Next.js 代理 404

**问题**：用户切换 Next.js 转发后仍提示“读取模型失败：404”。实际部署平台为 Zeabur。通过 curl 验证发现 `/ai-proxy/health` 返回 HTML 404 页面，而 `/webdav-proxy`（405）和 `/api/prompts`（200）正常，说明 Next.js 服务在运行。

**根因**：App Router 的 `app/ai-proxy/route.ts` 只能匹配 `/ai-proxy` 这一层路径，无法匹配 `/ai-proxy/models` 或 `/ai-proxy/health` 等子路径，因此拉模型请求 `/ai-proxy/models` 直接命中 404 页面。

**修复**：将 `app/ai-proxy/route.ts` 改为 `app/ai-proxy/[[...path]]/route.ts` 可选 catch-all 路由，同时支持 `/ai-proxy`、`/ai-proxy/models`、`/ai-proxy/chat/completions` 等所有子路径。健康检查改为直接访问 `/ai-proxy`（不带 `/health`），已推送 commit `d44f530`。

**验证步骤**：
1. 在 Zeabur 控制台重新部署
2. 浏览器访问 `https://你的域名/ai-proxy`，应返回 `{"ok":true,"timestamp":"...","mode":"ai-proxy"}`
3. 如 `/ai-proxy` 正常，再在配置里用 Next.js 转发拉取模型

**注意**：sub2api 当前只返回 `gpt-image-2` 图片模型，文本模型下拉框会为空。需在 sub2api 后台添加至少一个文字模型（如 `gpt-4o`）后再拉取。
