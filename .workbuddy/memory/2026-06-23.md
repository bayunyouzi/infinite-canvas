## 增加 AI 请求同源代理解决 CORS

**问题**：sub2api 部署后浏览器直接请求 `https://ai.bayunzi.shop/v1/models` 触发 CORS 预检 403，前端提示“读取模型失败”。

**修改文件**：
- `web/src/app/ai-proxy/route.ts`：新增 Next.js API 路由，透明转发 GET/POST AI 请求
- `web/src/stores/use-config-store.ts`：AiConfig 增加 `proxyMode`，`buildApiUrl` 支持 nextjs 代理
- `web/src/services/api/image.ts`：`aiApiUrl` / `aiHeaders` / `fetchImageModels` / `fetchChannelModels` 支持代理模式
- `web/src/services/api/video.ts`：视频请求支持代理模式
- `web/src/services/api/audio.ts`：音频请求支持代理模式
- `web/src/components/layout/app-config-modal.tsx`：渠道 Tab 增加连接方式切换（前端直连 / Next.js 转发）

**使用方式**：配置弹窗 → 渠道 → 连接方式 → 选择“Next.js 转发”，拉取模型和后续 AI 请求都会走当前站点服务端转发，绕过浏览器跨域限制。

## 排查 Next.js 代理 404

**问题**：用户切换 Next.js 转发后仍提示“读取模型失败：404”。

**可能原因**：Render `autoDeployTrigger: off`，代码推送后未自动重新部署，导致 `/ai-proxy` 路由不存在。

**修复**：在 `web/src/app/ai-proxy/route.ts` 增加 `/ai-proxy/health` 健康检查端点和更清晰的 JSON 错误响应，已推送 commit `ac285ad`。

**验证步骤**：
1. 在 Render 后台手动重新部署
2. 浏览器访问 `https://你的域名/ai-proxy/health`，应返回 `{"ok":true,"timestamp":"..."}`
3. 如返回 404，说明路由未生效，需检查 Render 部署日志或是否走了缓存域名
4. 如 health 正常，再在配置里用 Next.js 转发拉取模型
